# Review Status

Show the current status of the literature review.

## Steps

1. Confirm which review you're working on. If unclear, list all review directories at the root level and ask.

2. Read the review's files:
   - `References/references.yaml`
   - `References/reading-queue.yaml`
   - `References/subtopics.yaml`

3. Calculate and report:

   ### Overview
   - Review name and topic
   - Date created and last updated

   ### References
   - Total references: X
   - Read: X / Unread: X
   - By source: arxiv (X), web (X), blog (X), etc.
   - By priority: high (8-10): X, medium (5-7): X, low (1-4): X

   ### Reading Queue
   - Papers in queue: X
   - Next up: <highest priority paper>
   - Recently completed: <last 3 read>

   ### Subtopics
   - List each subtopic with:
     - Number of references
     - Number of key findings
     - Open questions count

   ### Coverage Assessment
   - Well-covered areas
   - Gaps or underexplored areas
   - Suggested searches to fill gaps

   ### Recent Activity
   - Last 5 papers added
   - Last 5 papers read

4. Suggest next actions:
   - Papers to read
   - Searches to run
   - Subtopics needing attention
   - Whether it's time to synthesize
